# Project Roadmap

> «Заявка на покупку» — веб-приложение для приёма и проверки заявок с ролевой авторизацией, сортировкой и пагинацией списка.

## Milestones

- [x] **Миграции и сидеры БД** — Prisma-схема (`User` с ролью, `PurchaseRequest` со статусом и автором, enum'ы `UserRole`/`RequestStatus`), первая миграция, seed-скрипт `prisma/seed.ts` с тестовыми пользователями (заявитель, проверяющий) и демо-заявками разных статусов для проверки списка, сортировки и пагинации.
- [x] **Общие контракты API на Zod** — общий пакет схем (`packages/contracts` или `shared/contracts`), переиспользуемый бэкендом и фронтендом: схемы `RegisterInput`/`LoginInput`, `CreateRequestInput`, `ListRequestsQuery` (поля сортировки, направление, пагинация), DTO ответов. Выведенные `z.infer`-типы — единый источник правды для обеих сторон. _Примечание: заменяет `class-validator` из `DESCRIPTION.md`; на бэкенде подключается через `nestjs-zod` (ZodValidationPipe)._
- [x] **Backend API на NestJS** — модули `auth`, `users`, `requests` по слоям `controllers → services → repositories`. Авторизация (register/login, JWT + Passport, bcrypt/argon2, `JwtAuthGuard` + `RolesGuard`), геттер таблицы заявок с сортировкой и пагинацией (`orderBy`/`skip`/`take` в репозитории), создание заявки заявителем. Валидация входа через общие Zod-контракты, единый формат ошибок (exception filter).
- [x] **Управление сессиями** — жизненный цикл сессии поверх JWT: access + refresh-токены, срок жизни и продление сессии, эндпоинт `refresh`, разлогин (инвалидация refresh-токена), защита от использования протухшего токена. На фронтенде — сохранение сессии между перезагрузками, автоматическое продление через интерцептор и корректный выход по истечении сессии.
- [x] **Frontend на Vue 3** — слои `api → stores (Pinia) → views/components`. Экран авторизации, форма создания заявки, таблица заявок с сортировкой по колонкам и пагинацией, skeleton-loading во время загрузки. Типизация вызовов через общие Zod-контракты, JWT-интерцептор, guard маршрутов по ролям.
